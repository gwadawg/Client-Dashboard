# Setter Daily Checklist

**Draft** — canonical setter daily workflow (replaces the former Operations Playbook). Same priority stack every day; calendar appointments only change *when* you pause for a live intro call.

## Purpose

Maximize qualified opportunities and demo bookings through a single daily priority list.

## Scope

Setter role: intro calls, speed-to-lead, confirmations, outbound, LinkedIn handoffs, no-shows, and EOD reporting. Deep execution lives in linked SOPs and scripts — not duplicated here.

## Quality bar

- Waiz voice: qualified conversations, acquisition system, strategic partner ([Identity Core](../../company/doctrine-identity-core-april-26.md)).
- Pricing and deal structure → escalate to Gabriel when applicable.
- Not DFY-qualified → Boot Camp, not demo ([Disqualifying and Financial Qualification](/library/financial-qualification)).

## How to use this doc

1. **Start of shift** — open **DQ-1** (Clear Today) to see today's outbound workload. See [GHL Daily Queue](ghl-pipeline-disposition-reference.md#smart-list-quick-reference).
2. **Always first** — true drop-everything items only.
3. **Priority list (P1–P7)** — work in order; do not drop to a lower priority until the one above is clear for this block.
4. **Intro calls (P1)** — any time you’re on an intro (alert, watchshift, or calendar slot), run the [intro script](/library/intro-call-script); don’t duplicate steps here.
5. **End of day** — submit EOD when you wrap (no fixed clock-out time). Update stages on the **Opportunities board** after every touch; use smart lists to find who to work.

## Start of Shift

- [ ] Confirm watchshift channels are live (GHL, Slack notifications on)
- [ ] Open **Contacts → Smart Lists** — pinned: **DQ-1**, **DQ-2**, **DQ-3**
- [ ] Glance at **DQ-1** queue size for the day
- [ ] **GHL tasks:** When you see a follow-up or opportunity, create a **task for yourself** in GHL. If it needs Gabriel, create a **task assigned to Gabriel** — don’t rely on memory or Slack alone.

---

## Always First (Drop Everything)

Only these override the current priority block. Everything else stays in the stack.

- [ ] **Watchshift alert** (Slack/GHL) → handle immediately per [Watchshift SOP](/library/watchshift), then ✅ or tag Gabriel on the thread
- [ ] **New inbound lead** → appears in **DQ-1**; call within watchshift window
- [ ] **New inbound booking** (notification) → [Priority 1](#priority-1--intro-calls) — call now, don’t wait for the slot
- [ ] **Live intro or demo no-show** (you’re on the call slot now) → [Priority 6](#priority-6--no-shows) protocol

---

## Texting Rules (All Priorities)

- **Value + context:** [Setter Lead Messaging](setter-lead-messaging.md) — read the lead file, form, and past texts before every SMS; notes on every lead you speak with.
- Prefer **call first** when you’re in a calling block; then text if no answer or **no SMS on file yet**.
- **Text replies do not stop outbound.** If someone texts back while you’re dialing, reply, then **jump back into outbound** — you don’t need to pause the whole block for a call unless it’s worth a live conversation now.
- Objection via text → steer toward booking; no long paragraph threads ([Objection Handling Hub](/library/objection-handling-hub))

## Priority 1 — Intro Calls

All intros use the same script and framework — **how** to run the call is only in those docs, not repeated here.

**When P1 applies (call or take the live intro now):**

- Watchshift alerts (SMS, new lead, new booking, no-show notification) → [Watchshift SOP](/library/watchshift)
- New inbound booking notification
- **Calendar intro slot is due** → pause P4–P7 until the call ends, then resume the list

**Checklist (priority doc only):**

- [ ] Intro handled per [Intro Call Script](/library/intro-call-script) — all [5 call checklist](/library/intro-call-script#call-checklist-5-things) items hit before hang-up
- [ ] CRM outcome logged; GHL task for self or Gabriel if follow-up remains

_Framework: [Intro Call Qualification Framework](/library/intro-qualification-framework) | Boot Camp routing: [Disqualifying and Financial Qualification](/library/financial-qualification)_

## Priority 2 — Watchshift

Live monitoring block **and** any watchshift alerts that fire during the day. **Cadence, alert types, Slack reactions, and call paths:** [Watchshift SOP](/library/watchshift).

- [ ] Watchshift channels active (GHL, Slack)
- [ ] Every watchshift item handled **immediately** — then ✅ on the Slack notification, or **tag Gabriel** if it needs him
- [ ] Calls and SMS run through [Intro Call Script](/library/intro-call-script) (opening type per situation)

_Framework: [Intro Call Qualification Framework](/library/intro-qualification-framework)_

## Priority 3 — Confirm Demo Appointments (Closer Calls)

Call prospects with a **demo** on the calendar in the next **24–48 hours** and **every demo morning-of**. Do not rely on automations alone.

**List:** Open **DQ-2 — Confirmations**. Work top to bottom. Includes intros and demos due within 2 days.

**Script:** [Demo Appointment Confirmation Script](script-demo-appointment-confirmation.md) — phone flow, content tie-down, SMS backup, CRM note template.

- [ ] Cleared **DQ-2** — every contact called (demos in next 24–48 hours **and today**)
- [ ] Called every booked prospect to confirm (word-for-word: confirmation script)
- [ ] Double tie-down: date/time + verbal commitment + computer/Zoom
- [ ] Pre-call video: re-tie watch commitment or send correct asset ([Pre-Call Objection Videos](../marketing/pre-call-objection-videos.md#quick-send-by-objection))
- [ ] Attempted to move demo earlier if a sooner slot is available (within 24hrs ideal)
- [ ] CRM note per script template (confirm + video + watch Y/N)
- [ ] Unresponsive 24hrs out → WhatsApp text + voice note, Instagram DM per SOP

_SOP: [No Shows and Maximizing Show Rates](/library/show-rate-levers)_

## Priority 4 — LinkedIn (Outreach Only)

**Replies: do not respond.** Forward everything to Gabriel.

- [ ] All new LinkedIn replies **forwarded to Gabriel** (no setter replies for now)
- [ ] GHL or internal task created if Gabriel needs context on a specific thread

**New outreach (30–60 min block when P1–P3 are clear)**

- [ ] 5–10 ICP post comments done first
- [ ] Mined lists, groups, or engagement for prospects
- [ ] ~90 seconds research per prospect before connecting
- [ ] Connect requests sent (blank or signal micro-note — A/B per copy-angles)
- [ ] First DM sent to accepted connections
- [ ] One no-reply bump at 48–72 hours — then stop

_Process: [LinkedIn Outbound Process](../outbound/linkedin/process.md) | Copy: [LinkedIn Copy Angles](../outbound/linkedin/copy-angles.md) | Compliance: [LinkedIn Compliance](../outbound/linkedin/compliance.md)_

## Priority 5 — Power Dialer: Clear DQ-1

Only after P1–P4 are clear for this block. **DQ-1** is also the P6 no-show queue — same list, same discipline.

**List:** **DQ-1 — Clear Today**. Work top to bottom until empty or every remaining row snoozed with **Next Work Date**.

- [ ] **DQ-1** worked top to bottom — New Lead, Engaged, Quality, No-Shows
- [ ] Within list: fresh New Leads and Engaged replies first, then A-grade, then rest
- [ ] **Touch today** = Last Human Touch updates (auto) → lead drops out for the day
- [ ] **Can't reach / defer** = set **Next Work Date** + GHL task + note
- [ ] Text after calling when needed; text replies per [texting rules](#texting-rules-all-priorities) — then back to DQ-1
- [ ] **Lead Grade** on every connect
- [ ] **Notes + pipeline stage** on every lead spoken with ([Setter Lead Messaging](setter-lead-messaging.md))
- [ ] GHL task for callbacks you can't hit today

_SOP: [Power Dialer New Leads SOP](sop-power-dialer-new-leads.md) | Lists: [GHL Daily Queue](ghl-pipeline-disposition-reference.md#setter-lists-daily-queue-model)_

## Priority 6 — No-Shows

No-shows in **Intro No Show** / **Demo No Show** stages appear in **DQ-1**. Work them there during P5/P6 using the protocol below. After DQ-1 is clear, archived no-shows can be pulled from the Opportunities board.

**Per no-show:**

- [ ] Waited 3 min — then called
- [ ] No answer → double-dialed immediately
- [ ] No answer → sent first text
- [ ] Called front desk (business hours) — “they booked a client generation call”
- [ ] **7 min past:** 3rd call from GHL + personal phone + Jeremy Miner text:

  > Hey it's [NAME] from [COMPANY], we had a call for [TIME] to get you more [CLIENT TYPE], tried to call you a few times and never heard back… where should we go from here?

- [ ] **10 min past:** final reschedule text:

  > Hey [NAME], called a few times. Are you interested in rescheduling?

- [ ] Lead moved to no-show pipeline stage in CRM
- [ ] Archived no-shows worked after this week’s list is done

_SOP: [No Shows and Maximizing Show Rates](/library/show-rate-levers)_

- [ ] Training completed today (roleplay, recording review, or assigned module)

## End of Day (When You Wrap)

Submit when you’re done for the day — flexible timing.

**Admin**

- [ ] Setter KPI tracker filled out
- [ ] **DQ-1 cleared** — zero rows, OR every remaining row has **Next Work Date** ≥ tomorrow + GHL task
- [ ] **DQ-3 cleared** — zero overdue scheduled follow-ups
- [ ] **Pipeline cleared** on Opportunities board — scan for orphans
- [ ] **Dispositions correct** — today's calls reflected in CRM
- [ ] Open **GHL tasks** up to date (your tasks + anything waiting on Gabriel)
- [ ] **Hotlist** — A-grade rows in DQ-1 touched or snoozed; see [EOD Report SOP](eod-report-sop-setters-closers.md)

**Metrics to report**

- [ ] Total dials made
- [ ] Total conversations
- [ ] Booked demos (from outbound)
- [ ] Calls confirmed

**Roll-up**

- [ ] Full call roll-up — every call: name, outcome, next step
- [ ] Objections / misaligned expectations noted

- [ ] **EOD form submitted** to sales channel

_SOP: [EOD Report SOP](eod-report-sop-setters-closers.md)_
